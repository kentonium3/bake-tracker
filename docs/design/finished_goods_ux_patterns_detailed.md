## Pattern Comparison

### Pattern A: Hierarchical Drawer

**Strengths:**
- Compact: main view stays clean, drawer is contextual
- Familiar: similar to file explorers, email clients
- Powerful filtering: all controls visible at once
- Multi-select: can add multiple components in one drawer session
- Flexible: works for simple and complex assemblies equally well

**Weaknesses:**
- Requires horizontal space (2-column layout)
- Filter combinations might be overwhelming at first
- User needs to understand what filters to apply
- Less guided - user needs to know what they're building

**Best for:**
- Power users who know what they want
- Creating complex assemblies with many components
- Users comfortable with filtering/searching patterns
- Desktop apps with wide screens

**Paper prototype test:**
- Draw split screen on paper
- Use sticky notes for filter buttons
- Use index cards for component list items
- Walk through: "Find chocolate cake, add it, then find box"

---

### Pattern B: Accordion Builder

**Strengths:**
- Highly guided: user knows exactly what to do next
- Progressive disclosure: only see relevant options
- Natural flow: base â†’ packaging â†’ extras â†’ review
- Easier for beginners: less overwhelming
- Validates as you go: can't skip required steps

**Weaknesses:**
- Linear flow: harder to jump back and change earlier choices
- More clicks: expand/collapse for each step
- Less flexible: assumes certain assembly order
- Might feel restrictive for power users
- Doesn't work well if user wants to add multiple of same type

**Best for:**
- First-time users learning the system
- Standard assembly patterns (base + packaging + extras)
- Guided workflows where order matters
- Touch-friendly interfaces (mobile)

**Paper prototype test:**
- Draw accordion sections on paper
- Use paper clips to show expanded/collapsed states
- Use index cards for radio button selections
- Walk through: "Build boxed cake" watching which steps feel natural

## Recommendation for Paper Prototyping

**Start with Pattern B (Accordion)** because:

1. **Easier to sketch:** Just draw 4 stacked boxes on paper
2. **Easier to walk through:** "Here's step 1, pick a cake. Now step 2, pick packaging..."
3. **Tests your mental model:** Does the order (base â†’ pkg â†’ extras) match Marianne's thinking?
4. **Reveals missing steps:** Will she say "wait, I need to do X before Y"?

**Then try Pattern A (Drawer)** if:
- Marianne finds accordion too restrictive
- She wants to jump around (add packaging before base, etc.)
- She needs to add multiple types of same component

**Test scenarios for paper prototypes:**

1. **Simple:** Individual cookie in cellophane
2. **Medium:** Boxed cake with ribbon
3. **Complex:** Gift basket (tray + 3 brownies + 4 cookies + tissue + cellophane)
4. **Editing:** Change an existing finished good (swap box type)

**Time investment:**
- Pattern B sketch: 30 minutes
- Marianne walkthrough: 30 minutes
- Pattern A sketch: 45 minutes (if needed)
- Marianne comparison: 20 minutes

**Total: ~2 hours to find the right approach before writing any code.**
